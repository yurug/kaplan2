What you are looking at. Every box is a cell in memory; the dots inside are the elements it holds (in a front buffer and a back buffer); the arrows are the links from one cell to the next. The layout is a live force simulation — drag any box, and use the buttons to add/remove elements and watch the graph rearrange. Each cell is tinted green / yellow / (orange /) red by how full it is.

Driven by the verified code. The two interactive tabs (Deque, Catenable deque) run the actual, tested implementation — a TypeScript port in ts/ of the machine-checked Rocq code — bundled into the page and rendered straight from the real structure, so what you see is the real memory after each step, not a hand-drawn approximation. In the Deque, an element one level down bundles 2× as many base values (a pair, then a pair-of-pairs, …); in the catenable deque, each entry is a small bundle (a triple) and they nest one level deeper each step down. A catenable deque stays a single fat cell under plain add/remove; the tree only appears when you join two of them (the ⋈ button). The two storyline tabs (persistence, concat) are curated step-throughs that illustrate sharing. The implementation is verified the same way as the C version: random testing against a simple reference, a worst-case-O(1) bound on work per operation, and a byte-for-byte trace match against the original (see ts/README.md).
